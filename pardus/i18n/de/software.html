    Verschiedene Reparatur Möglichkeiten (Fixes)

    Dieser Abschnitt enthält Vorgänge, die häufig auftretende Probleme beheben können sollte ein Fehler auftreten, während ein Upgrade oder eine neue Software installiert wird.

    Veraltete Paketlisten?

    Ihr Paketlisten können veraltet sein, was kann Nicht gefunden Fehler und veraltete Version Information dazu führen wenn sie versuchen, neue oder noch neuere Versionen der Software zu installieren. Dies ist insbesondere dann der Fall, wenn Ubuntu MATE online nach der Installation zum ersten Mal verbindet.

    Aktualisieren Sie die Paketlisten:
    sudo apt-get update
    Paketen Quelle-List aktualisieren Installiert Pakete aktualisieren

    Kaputte Pakete?

    Wenn eine vorherige Installation oder Entfernung unterbrochen wurde (ZB durch Stromausfall oder Verlust der Internetverbindung), weitere Software kann nicht ohne richtig neu konfigurieren diese kaputten Pakete, hinzugefügt oder entfernt werden. Falls erforderlich, müssen Sie auch die Abhängigkeiten auflösen und alle fehlenden Pakete installieren, so dass Sie die Software ordnungsgemäß ausgeführt werden können.

    Konfigurieren Sie Pakete, die entpackt war, aber noch nicht konfiguriert sind:
    sudo dpkg --configure -a
    Kaputten Abhängigkeiten herunterladen und installieren:
    sudo apt-get --fix-broken install
    Abgebrochen Pakete konfigurieren Kaputte Pakete auflösen

    Darüber hinaus, unten sind entsprechenden Terminal Befehle aufgelistet, die sonst in die Softwareaktualisierung Verwalter erscheinen.

    Aktualisieren Sie alle Pakete, die eine neue Version zur Verfügung haben:
    sudo apt-get dist-upgrade
    Zu eine neue Version von Ubuntu MATE aktualisieren:
    sudo do-release-upgrade

    Proprietärer Software ist Eigentum einen Person oder ein Unternehmen. Es kann Beschränkungen seiner Nutzung geben (wie eine Lizenzvereinbarung), und gibt es möglicherweise kein Zugriff auf den Quellcode.

    Freie und Open Source Software räumt der Benutzer die Freiheit die Software zu teilen, studieren und modifizieren ohne Einschränkung. Die GPL ist ein Beispiel für eine "Freie Software" Lizenz.
