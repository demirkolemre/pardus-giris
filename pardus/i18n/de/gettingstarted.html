Installation wird vorbereitet

Ubuntu MATE ist relativ einfach einzurichten, ob Sie sich Ihren vorhandenen Betriebssystem ersetzen, oder Ubuntu MATE neben einem bestehenden installieren möchten  wie Microsoft Windows oder ein anderes GNU/Linux-Distribution.

Für die besten Ergebnisse:

  • Sichern Sie alle wichtigen Daten, einen andere Speichergerät auf.
  • Mit dem Internet eine Verbindung herstellen um alle verfügbaren Aktualisierungen während der Installationsvorgang zu installieren.
  • Um alle verfügbaren Aktualisierungen zu installieren, bleiben Sie mit dem Internet verbunden.
  • Falls Ihr Gerät im Akkubetrieb läuft, schliessen Sie es das Stromnetz jetzt an.

Die derzeitige Live- Sitzung ist jetzt eine gute Gelegenheit Ubuntu MATE zu testen ohne den Computer zu modifizieren, so dass Sie alle Peripheriegeräte testen können, bevor Sie zu die Installation auf Ihrem Gerät begehen. Wenn Sie bereit sind, doppelklicken Sie auf das Installationssymbol auf dem Desktop.

Moderne UEFI -basierten Computern

Wenn Sie einen modernen PC haben, der die Einheitlich erweiterten Firmware Interface (UEFI) verwendet, müssen Sie das Secure Boot deaktivieren (oder einen Bootloader freischalten (unlock) Option aktivieren) vor der Installation, insbesondere wenn ein Betriebssystem vorinstalliert wurde.

Wenn Sie eine Legacy- BIOS-Maschine (Gerät) verwenden, brauchen Sie dies nicht zu tun.

Vor allem, wenn Sie Dual-Booten, ist es wichtig, dass Sie Ihr System im richtigen Modus starten und verwenden Sie die 64-Bit Abbild, da Sie nicht ein installierte Betriebssystem im BIOS -Modus starten können, nachdem ein UEFI -fähiges Betriebssystem oder umgekehrt installiert war.

Booten im BIOS -Modus auf einem UEFI -System mit einer GPT - formatierten Festplatte, wird auch eine "Schutzschicht" zeigen. Keine Sorge, Ihre Daten ist nicht verloren gegangen oder beschädigt!

Swap (Auslagerungspartition)

Bei Computern die plötzlich wenig Speicher (RAM) haben, wird das Betriebssystem auf der Auslagerungspartition (Swap) zugreifen. Bekannt als "Auslagerungsdatei" in Windows. Die Leistung wird abnehmen, wenn wenig Speicher vorhanden ist und könnte möglicherweise einfrieren wenn Sie völlig ohne irgendwelche Platz zum auslagern haben.

Standardmäßig, wird eine Swap-Partition basiert auf der Größe Ihrer Arbeitsspeicher (RAM) erstellt. Zum Beispiel, wenn Sie 2 GiB RAM haben, dann wird 2 GiB Festplattenspeicher für Swap am Ende der Laufwerk reserviert werden.

Dual- Booten (zwei Betriebssystemen auf den selben PC verwenden)

Möchten Sie Ubuntu MATE neben Ihrer bestehenden Betriebssystem installieren, müssen Sie die Partitionen schrumpfen. Dies führt zu Aufteilung in den Abschnitten der Festplatte zwischen mehreren Betriebssystemen.

Um das Risiko von Datenverlust oder Fehler zu minimieren, stellen Sie sicher, dass:
  • Es gibt genügend freier Speicherplatz. 10 GB Minimum ist empfohlen.
  • Sichern Sie wichtige Daten auf ein anderes Laufwerk kopieren.
  • Defragmentieren Sie der Festplatte.
  • Stellen Sie sicher dass Windows ordnungsgemäß heruntergefahren (nicht in Ruhezustand gesetzt) wurde.

Sie können diese Einhängepunkte, spezifisch auswählen:

  • / - Die Basis für Ubuntu MATE
  • /boot - Bootloader und Kernel
  • /home - Ihre persönlichen Dateien und Ordner
  • /tmp - Temporäre Dateien
  • /usr - Enthält die meisten der Anwendungen (Binärdateien, Dokumentation, Bibliotheken, etc.)
  • /var - Variable Daten (Wie zum Beispiel System-Log Dateien)
  • /srv - Daten für Systemdienste (typischerweise für Serverkonfigurationen verwendet)
  • /opt - Zusätzliche Software nicht vom Paketmanager verwaltet (sowie Spotify)

Erstellen von separaten Partitionen für alle oben angezeigten Einhängepunkten ist nicht erforderlich. Die allgemein partitioniert Verzeichnisse sind /, /home und /boot. Diese Einhängepunkte können auch über verschiedene physikalische Laufwerke eingestellt werden, zum Beispiel, wenn Sie das Betriebssystem / auf einer SSD, aber Ihre persönlichen Dateien /home auf einer mechanischen Festplatte.

Reparieren von Problemen beim Start (Booten)

Linux-Distributionen verwenden einen Bootloader (Startprogramm) bekannt alsgrub, die nach das Betriebssystem sucht und das Betriebssystem startet. Allerdings integrieren einige Betriebssysteme (zB Windows) nicht mit den bestehenden Bootloader und wird sie überschreiben, mit dem Potenzial, den Zugriff auf Ubuntu MATE zeitlich zu verlieren.

Glücklicherweise kann man das Boot-Reparatur Tool verwenden um häufige Probleme zu beheben, welche durch andere Betriebssysteme verursacht werden.

Proprietäre oder Open Source Treiber?

Proprietäre Treiber kommen direkt vom Hersteller. Dadurch dass sie privaten Quelltext sind, können die Ubuntu-Entwickler weder überprüfen noch verbessern können was bedeutet dass, Sicherheit- und andere Aktualisierungen hierfür können nur vom Treiber-Anbieter bereitgestellt werden.

Ferner, können proprietäre Treiber die Grafik-Leistung signifikant erhöhen und im Ergebnis, das ganze System. Benutzer sollten diese Treiber installieren zum:

  • 3D-Anwendungen und Spiele ausführen.
  • Compiz -Fenstereffekte Verwenden
  • Die Energieverwaltung verbessern, vor allem, wenn das Gerät mehr als üblich unter die Open Source (quelloffen) Treibern überhitzt.
  • Verbessert die Unterstützung für hochauflösende und mehrere Monitore.
  • leere Bildschirm Probleme beseitigen.

Proprietäre Treiber werden als "binäre Blobs" bekannt. Mit bestimmte Hardware kann noch schlimmerer Verhalten auftreten, mehr Probleme verursachen dann die Open Source Treiber, sowie:

  • In den grafischen Desktop zu starten.
  • Defekte Treiber nach einem Upgrade.
  • Schwere Grafikfehler.
  • Probleme mit Anwendungen/Spielen welche die Hardware-Beschleunigung benötigen.

GParted

Ein erweitertes Tool zum Anzeigen, Erstellen, Ändern und Löschen von Partitionen. Es hat auch Funktionen Dateisysteme zu erweitern/verkleinern und unterstützt eine breite Palette von Formaten.

Festplattennutzung Analysator

Systemmonitor

Zeigt aktuell laufenden Prozesse, die Ressourcennutzung und die Menge verwendete Speicherplatz für frei und eingehängte Dateisysteme an.

Psensor

Zeigt in grafische Form, die aktuelle Temperatur der angeschlossen System Komponenten an. Inklusive : Motherboard, CPU, GPU und Festplatten.
