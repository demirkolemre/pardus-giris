Requisitos del Sistema Modestos

Ubuntu MATE funciona bien tanto en ordenadores de calidad superior como en ordenadores lentos, está optimizado para ser funcional, al mismo tiempo que es ligero con respecto a recursos del sistema.

Mínimos
  • Pentium III 750 megahercios (MHz)
  • 512 megabytes (MB) de RAM
  • 8 gigabytes (GB) de espacio en disco
Recomendados
  • Core 2 Duo 1.6 gigahercios (GHz)
  • 2 gigabytes (GB) de RAM
  • 16 gigabytes (GB) de espacio en disco

El sistema operativo se ejecutará de una forma mucho más eficiente si tu ordenador reúne o sobrepasa los requisitos recomendados.

Preparando para Instalar

Ubuntu MATE es relativamente sencillo de instalar, si estás buscando reemplazar tu sistema operativo actual o instalar Ubuntu MATE junto con uno presente - como  Microsoft Windows u otra distribución GNU/Linux.

Para los mejores resultados:

  • Realiza una copia de seguridad de cualquier dato importante en otro dispositivo de almacenamiento.
  • Conectate a Internet durante la instalación para instalar todas las actualizaciones disponibles.
  • Mantente conectado a Internet para instalar todas las actualizaciones disponibles.
  • Enchufa tu dispotivo a la red eléctrica si está funcionando con batería.

La sesión en vivo en la que estás ahora mismo es una buena para probar Ubuntu MATE sin modificar tu ordenador, permitiéndote comprobar que todos tus periféricos estás funcionando como debieran antes de llevar a cabo la instalación en tu máquina. Una vez estés listo, haz doble click sobre el icono de instalación del escritorio.

Ordenadores Modernos basados en UEFI

Si tienes un PC moderno que utiliza la Interfaz de Firmware Extensible Unificada (UEFI), puede que necesites deshabilitar Inicio Seguro (Secure Boot) (o habilitar una opción Desbloquear Gestor de Arranque (Unlock Bootloader) ) antes de la instalación, particularmente if un sistema operativo fue preinstalado.

Si estás usando una máquina con la BIOS tradicional, no necesitas hacer esto.

Especialmente con arranque dual, es importante que inicies tu sistema del modo correcto y uses la imagen de 64 bits , ya que no puedes iniciar un sistema operativo instalado en modo BIOS después de instalar un sistema operativo con UEFI habilitado o viceversa.

Iniciar el sistema en modo BIOS en un sistema UEFI con un disco con formato GPT también mostrará una «ċapa de protección». No se preocupe, ¡sus datos no se han perdido ni corrompido!

Intercambio

Para ordenadores que empiecen a funcionar con poca memoria (RAM), el sistema operativo empezará a intercambiar a disco. Conocido como "archivo de paginación" en Windows. El rendimiento se reducirá mientras se trabaje con memoria baja y podría dejar de responder si agota toda la memoria sin ningún sitio donde intercambiar.

De forma predeterminada, se crea una partición de intercambio basado en el tamaño de la memoria RAM. Por ejemplo, si usted tiene 2 GB de memoria RAM, de 2 GB de espacio en disco duro se reservará para intercambio al final de la unidad.

Usando la opción "Personalizada" del instalador.

En su lugar o adicional a Gparted, la opción "Personalizada" durante el asistente de instalación también puede configurar y cambiar el tamaño de las particiones, así como elegir los puntos de montaje. La división de los discos puede hacer que sea un poco más fácil volver a instalar el sistema operativo posterior si así lo desea.

Puedes especificar éstos puntos de montaje:

  • / - La base para Ubuntu MATE
  • /boot - Gestor de arranque y los kernels
  • /home - Tus archivos y carpetas personales
  • /tmp - Archivos temporales
  • /usr - Contiene la mayoría de las aplicaciones (binarios, documentación, bibliotecas, etc.)
  • /var - Datos variables (Tales como los registros del sistema)
  • /srv - Datos para Servicios del Sistema (usualmente usado para configuraciones de servidor)
  • /opt - Software adicional no administrado por el administrador de paquetes (así como Spotify)

La creación de particiones separadas para todos los puntos de montaje de arriba no es necesario. Los directorios comúnmente con particiones son /, /home y /boot. Estos puntos de montaje también pueden ser configurados a través de diferentes discos físicos, por ejemplo, si prefieres el Sistema Operativo / en una Unidad de estado sólido pero tus archivos personales /home en un Disco duro mecánico

¿Controladores propietarios o de código abierto?

los controladores propietarios vienen directamente del fabricante. Como no son de código abierto, los desarrolladores de Ubuntu son incapaces de revisar y mejorar el código Lo que significa que debes confiar en el fabricante para soporte técnico.

Además, los controladores propietarios pueden mejorar el rendimiento gráfico significativamente y consecuentemente, el sistema entero. Los usuarios pueden preferir instalar estos controladores para:

  • Correr aplicaciones y juegos en 3D.
  • Usar efectos de ventana a través de Compiz.
  • Mejorar la administración de energía, sobre todo si el dispositivo experimenta problemas de sobrecalentamiento con controladores de código abierto.
  • Mejorar el apoyo a altas resoluciones y varios monitores.
  • Solucionar problemas de pantalla en blanco.

Los controladores propietarios son conocidos como "blob binario". Determinado hardware puede experimentar peor comportamiento, causando más problemas que los controladores de código abierto, tales como:

  • El no poder arrancar en el escritorio gráfico.
  • Controladores rotos tras una actualización.
  • Fallos gráficos graves.
  • Problemas con aplicaciones y/o juegos que requieren la aceleración de hardware.

¿Unidades SI o IEC?

Decimal (Estántar SI)

  • 1 KB = 1000 bytes
  • Pronunciado como "ga" - como "Mega", "Giga".
  • Común en aplicaciones de uso diario y almacenamiento de archivos.

Binario (Estándar IEC)

  • 1 KiB = 1024 bytes
  • Pronunciado como "bi" - como "Mebi", "Gibi".
  • Común en la memoria RAM, las unidades y particiones, incluyendo herramientas como GParted.
