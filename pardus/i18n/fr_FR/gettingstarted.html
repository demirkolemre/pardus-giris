Exigences Systèmes Modestes

Ubuntu MATE fonctionne aussi bien sur des ordinateurs puissants ou moins puissants. Il est optimisé pour être fonctionnel et exiger peu de ressources systèmes.

Minimum
  • Pentium III 750 megahertz (MHz)
  • 512 megabytes (MB) de mémoire vive (RAM)
  • 8 gigabytes (GB) d'espace disque
Recommandé
  • Core 2 Duo 1.6 gigahertz (GHz)
  • 2 gigabytes (GB) de mémoire vive (RAM)
  • 16 gigabytes (GB) d'espace disque

Le système d'exploitation fonctionnera de manière plus efficace si votre ordinateur dépasse les exigences relatives au matériel.

Préparations à l'installation

Ubuntu MATE est relativement simple à installer et configurer, que vous cherchiez à remplacer votre système d'exploitation ou que vous installiez Ubuntu MATE aux côtés d'un système existant comme  Microsoft Windows ou une autre Distribution GNU/Linux.

Pour de meilleurs résultats :

  • Faites une sauvegarde de toutes vos données importantes sur un autre appareil de stockage.
  • Connectez-vous à Internet afin d'installer toutes les mises à jour disponibles pendant l'installation.
  • Restez connecté(e) à Internet afin d'installer toutes les mises à jour disponibles.
  • Branchez votre appareil à une source d'alimentation électrique si la batterie est faible.

La session Live dans laquelle vous êtes en ce moment est une bonne occasion de tester Ubuntu MATE sans modifier votre système, ce qui permet de tester vos périphériques et vérifier qu'ils fonctionnent comme ils le doivent avant de décider si oui ou non vous voulez installer Ubuntu MATE sur votre machine. Une fois prêt(e), double-cliquez l'icône d'installation sur le bureau.

Ordinateurs modernes basés sur UEFI

Si vous avez un PC moderne utilisant l'Interface Firmware Étendue Unifiée (UEFI), il vous sera peut-être nécessaire de désactiver le Démarrage Sécurisé (Secure Boot) (ou d'activer l'option débloquant le Chargeur d'Amorçage ) avant installation, en particulier si vous aviez déjà un système d'exploitation installé auparavant.

Si vous utilisez une machine fonctionnant à partir d'un BIOS legacy, vous n'aurez pas besoin de faire ceci.

Spécifiquement quand vous utilisez un double démarrage, il est important que vous démarriez le système dans le mode approprié et que vous utilisiez l'image disque 64-bit , car vous ne pouvez pas démarrer un système d'exploitation installé en mode BIOS après avoir installé un système en mode UEFI et vice versa.

En démarrant en mode BIOS sur un système UEFI avec un disque formatté en mode GPT une "couche de protection" se manifestera, vous empêchant de continuer. Ne craignez rien, Vos données ne sont ni perdues ni corrompues !

Votre système fonctionne actuellement en ??? mode.

Mémoire Virtuelle (Swap)

Pour les ordinateurs qui commencent à manquer de mémoire vive (RAM), le système d'exploitation commencera à mettre les données en swap sur le disque. Cette mémoire est connue en tant que "mémoire virtuelle de pagination" sur Windows. Les performances diminuent lorsque la mémoire vive est presque toute utilisée et l'ordinateur pourrait possiblement geler s'il utilise complètement sa mémoire vive sans avoir d'endroit pour consigner les données en swap sur le disque.

Par défaut, une partition de swap est créée en fonction de la taille de votre mémoire vive (RAM). Par exemple, si vous avez 2 GiB de RAM, 2 GiB d'espace disque seront réservés comme swap à la fin de votre disque.

Double-Démarrage (Dual Boot)

Dans le cas ou vous voudriez installer Ubuntu MATE aux cotés de votre système d'exploitation, vos partitions auront besoin d'être rétrécies. Ce qui implique que des sections de votre disque dur seront divisées entre différents systèmes d'exploitation.

Afin de minimiser les risque de perte de données ou de panne, assurez-vous que :
  • Vous ayez assez d'espace disque libre. 10 GB minimum sont recommandés.
  • Faites une sauvegarde vos données importantes sur un autre disque.
  • Défragmentez le disque dur.
  • Assurez-vous que Windows a été arrêté convenablement (qu'il ne soit pas en mode hibernation)

Open GParted

Une fois les opérations sur vos disques complétées, vous pouvez laisser l'espace disque restant non-alloué. L'installeur détectera cet espace non-utilisé et y installera Ubuntu MATE.


Utilisez le mode "Manuel" de l'installeur.

À la place ou en plus de GParted, le mode Manuel de l'assistant d'installation peut aussi configurer et modifier la taille de vos partitions ainsi que choisir des points de montage. Diviser vos disques peut faciliter une éventuelle réinstallation du système d'exploitation ultérieure.

Vous pouvez spécifier ces points de montage :

  • / - La base d'Ubuntu MATE
  • /boot - Le Chargeur d'Amorçage (Bootloader) et les noyaux (kernels)
  • /home - Vos fichiers et dossiers personnels
  • /tmp - Les fichiers temporaires
  • /usr - Contient la plupart des applications (fichiers binaires, documents, librairies, ...etc)
  • /var - Données variables (comme des fichiers de journalisation système)
  • /srv - Données des Services Système (habituellement utilisées pour des configurations serveurs)
  • /opt - Les logiciels qui ne sont pas gérés par le gestionnaire de paquets (comme Spotify)

Il n'est pas nécessaire de créer des partitions pour chaque point de montage mentionné plus haut. Souvent les dossiers partitionés sont /, /home et /boot. Ces points de montage peuvent aussi s'étendre sur différents disques physiques. Par exemple si vous voulez que le système d'exploitation / soit sur un SSD, mais que vos fichiers personnels soient /home sur un disque dur mécanique.

Rapetisser votre disque système sous Windows.
Cela s'applique à Windows Vista et plus.

Si vous utilisez Windows XP ou un système plus ancien, il vous faudra utiliser la méthode Simple ou Avancée afin de rapetisser votre disque système.

Les nouvelles versions de Windows contiennent un utilitaire pour diminuer la taille du volume NTFS pendant que votre système est en marche. Ce qui est plus sûr s'il est essentiel que Windows puisse toujours démarrer sur votre ordinateur alors que vous faites l'expérience d'Ubuntu MATE.

1. Ouvrir Gestion des Disques à partir du menu de démarrage.

2. Faites un clic droit sur le disque désiré et sélectionnez Rétrécir Partition dans le menu contextuel.

3. Entrer la taille désirée pour votre partition système.

Souvenez-vous, Ubuntu MATE a besoin de 8 GB d'espace disque minimum. 1 GB = 1000 MB

Réparer des Problèmes de Démarrage

Les distributions Linux utilisent un chargeur de d'amorçage (boot loader) connu sous le nom degrub, qui recherche et démarre le système d'exploitation. Cependant, certains systèmes (comme Windows) ne s'intègrent pas à d'autres chargeurs d'amorçage et essayera de les remplacer, avec la possibilité de perdre temporairement accès à Ubuntu MATE.

Par chance vous pouvez utiliser l'outil de Réparateur d'Amorçage (Boot Repair) pour réparer les problèmes les plus courants causés par d'autres systèmes d'exploitation.

Pilotes Propriétaires ou Open Source ?

Les pilotes propriétaires proviennent directement du fabricant. Comme ils ne sont pas open source, les développeurs d'Ubuntu ne peuvent pas examiner ou améliorer le code ce qui signifie que vous devrez vous référer au fabricant pour le support.

De plus, les pilotes propriétaires peuvent grandement améliorer les performances graphiques et par extension le système tout entier. Les utilisateurs pourraient désirer installer ces pilotes pour :

  • Lancer des applications 3D et des jeux.
  • Utiliser les effets de fenêtres de Compiz.
  • Améliorer la gestion de l'alimentation, surtout si l'appareil surchauffe avec les pilotes open source.
  • Améliorer le support des hautes résolutions ainsi que des écrans multiples.
  • Résoudre les problèmes d'écrans noirs.

Les pilotes propriétaires sont connus sous le nom de "blobs binaires". Certains périphériques se comporteront moins bien, ce qui créera plus de problèmes qu'avec les pilotes open source. Des problèmes comme :

  • L'échec du démarrage de l'interface graphique.
  • Des pilotes cassés après une mise à jour.
  • Des bugs graphiques sévères.
  • Des problèmes avec les jeux et applications requérant une accélération matérielle.

Unités SI ou IEC ?

Décimal (Standard SI)

  • 1 KB = 1000 bytes
  • Prononcé "ga" comme "Mega", "Giga".
  • Il est souvent employé dans les applications de tous les jours et de stoquage de fichiers.

Binaire (Standard IEC)

  • 1 KiB = 1024 bytes
  • Prononcé "bi" comme "Mebi, "Gibi".
  • Il est souvent employé pour se référer à la taille de la Mémoire Vive (RAM), des disques durs et des partitions, ainsi que dans des outils comme GParted.
